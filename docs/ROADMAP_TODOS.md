## 1) 核心工作流（整理/去重/归档）

- [ ] **P0 按筛选批量操作能力（后端）**
  - 目标：支持"全选当前筛选结果"后执行批量操作，而无需把所有条目先加载到前端。
  - 方式：新增后端批量接口支持 `filterSpec`（当前筛选条件）+ `limit/cap` + `excludeIds` 等，服务端执行并写入 `file_ops`。
  - 状态：当前批量操作需要前端先加载所有选中项

- [ ] **P0 归档落盘一致性增强**
  - 目标：保证 `file_ops`、文件系统与 DB 在崩溃/中断后可恢复到一致状态。
  - 动作：完善 `/sync` 的对账策略与错误可观测性（错误列表、重试策略）。
  - 状态：基础 `/sync` 已实现，但错误可观测性和重试策略待完善

- [x] **P1 多选与选中集合管理（前端）** ✅
  - 目标：全选/反选/清空/查看已选中，并能在抽屉中快速剔除。
  - 取舍：全选有硬上限（例如 500），超过则强提示用户进一步筛选。
  - 状态：已实现（SelectedDrawer 组件支持查看已选中、清空、移除单个项）

## 2) UI/用户体验

- [ ] **P0 详情面板查看原图/原视频**
  - 目标：在详情面板中支持查看原高清图片和原视频（非缩略图），提供全屏/缩放等基础查看能力。
  - 方式：通过文件路径直接服务原文件，或提供专门的查看器组件。
  - 状态：当前详情面板仅显示缩略图，未实现原图/原视频查看功能

- [x] **P1 Folders/Archives 列表体验** ✅（部分）
  - 目标：文件夹列表增加计数、最近更新时间、可选封面；点进后分页/虚拟化浏览。
  - 状态：已实现计数、封面、虚拟化浏览（AlbumAssetsGrid）；缺少"最近更新时间"显示

- [ ] **P1 Tags 的 UI 与筛选**
  - 目标：先做手动标签（地点/主体/人物/食物）+ 基础筛选；绑定在 `assets(hash)` 上。
  - 未来：预留模型自动打标的写入路径与审核/纠错流程。
  - 状态：数据模型已预留（`tags`/`asset_tags` 表），但 UI 和筛选功能未实现

- [ ] **P1 视频与 RAW 格式缩略图策略**
  - 目标：对常见视频生成封面缩略图（或抽帧），对 RAW 格式（CR2/NEF/ARW 等）生成预览缩略图，避免大量"无缩略图"的体验落差。
  - 方式：best-effort（视频用 ffmpeg 抽帧，RAW 用 exiftool/raw-thumbnail-extractor），失败保持 `thumb_status='unsupported'`。
  - 状态：当前仅支持图片格式（sharp），视频和 RAW 格式未实现

- [x] **滚动与日期定位（date-index + overlay）** ✅
  - 目标：大库滚动时随机访问可视区域页；日期提示稳定不跳动；支持"跳到某月"。
  - 状态：已实现（`GET /files/date-index` API + FilesGrid 中的 overlay 日期显示和月份跳转）

## 3) 性能与稳定性

- [x] **P0 扫描吞吐与资源控制** ✅（部分）
  - 方向：队列并发可配置；更好的 backpressure；更清晰的进度/错误统计。
  - 状态：队列并发已实现（QUEUE_CONCURRENCY = 4），但固定不可配置；进度统计已实现，但 backpressure 和错误统计可进一步优化

- [x] **P0 大库 UI 性能** ✅（部分）
  - 方向：进一步减少无意义请求（例如页 overscan 上限）；列表滚动节流与缓存策略。
  - 状态：已使用虚拟列表（react-virtual）和 overscan 优化；滚动节流已实现（FilesGrid 中有 throttle）；但可能还有进一步优化空间

- [x] **增量更新（SSE + batch patch）** ✅
  - 目标：扫描/哈希/缩略图补全渐进更新，UI 不整页刷新。
  - 状态：已实现（`GET /changes/stream` SSE + 前端 batch API 拉取并 patch 已加载项）

## 5) AI/智能功能（需评估模型依赖与性能）

- [ ] **P1 物品种类识别**
  - 目标：接入图像识别模型，自动识别图片中的物品种类（如：动物、建筑、食物、车辆等），并可作为标签或筛选条件。
  - 方式：评估轻量级模型（如 CLIP/ResNet 等）的本地部署方案，或调用外部 API；识别结果写入 `tags` 或扩展的 `asset_tags`。
  - 取舍：需平衡识别准确率、计算成本与响应速度；可先做"后台异步识别 + 渐进展示"。
  - 状态：未实现

- [ ] **P1 "找人"功能（人脸识别与检索）**
  - 目标：支持识别照片中的人脸，建立人脸库，并可按人物查找所有相关照片。
  - 方式：
    - 预处理阶段：扫描时提取人脸特征并建立人脸库（`faces` 表：`id, hash, face_embedding, person_id`）。
    - 检索阶段：支持"标记某张照片中的人脸为某人"后，查找所有包含该人物的照片。
  - 取舍：需评估人脸识别模型（如 face-api.js/insightface）的依赖与性能；人脸库管理（合并/拆分/重命名）的交互复杂度。
  - 状态：未实现
