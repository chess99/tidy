## 1) 已明确（P0/P1）TODO

### P0：核心工作流完善（整理工具主链路）

- **按筛选批量操作能力（后端）**
  - 目标：支持“全选当前筛选结果”后执行批量操作，而无需把所有条目先加载到前端。
  - 方式：新增后端批量接口支持 `filterSpec`（当前筛选条件）+ `limit/cap` + `excludeIds` 等，服务端执行并写入 `file_ops`。
- **多选与选中集合管理（前端）**
  - 目标：全选/反选/清空/查看已选中，并能在抽屉中快速剔除。
  - 取舍：全选有硬上限（例如 500），超过则强提示用户进一步筛选。
- **归档落盘一致性增强**
  - 目标：保证 `file_ops`、文件系统与 DB 在崩溃/中断后可恢复到一致状态。
  - 动作：完善 `/sync` 的对账策略与错误可观测性（错误列表、重试策略）。

### P1：整理效率提升（依然是“整理工具”范畴）

- **视频缩略图策略**
  - 目标：对常见视频生成封面缩略图（或抽帧），避免大量“无缩略图”的体验落差。
  - 方式：best-effort（ffmpeg 抽帧），失败保持 `thumb_status='unsupported'`。
- **Tags 的 UI 与筛选**
  - 目标：先做手动标签（地点/主体/人物/食物）+ 基础筛选；绑定在 `assets(hash)` 上。
  - 未来：预留模型自动打标的写入路径与审核/纠错流程。
- **Folders/Archives 列表体验**
  - 目标：文件夹列表增加计数、最近更新时间、可选封面；点进后分页/虚拟化浏览。

## 2) 潜在（需要评估价值/复杂度）TODO

### 半自动归类（不做完整相册）

- **旅程建议（time+location）**
  - 目标：基于时间范围 +（未来可选）地点标签，建议 `YYYYMMDD-地点` 文件夹候选。
  - 取舍：默认只做“建议 + 人工确认”，不做全自动落盘。
- **相似内容聚类（去重/近重复）**
  - 目标：辅助发现“近重复”（连拍、截图重复、不同分辨率同内容）。
  - 方式：感知哈希/视频指纹（需评估依赖与性能）。

### 更强的“复活”治理

- **外部变更监控（watcher）**
  - 目标：监听 WORK_ROOT 的增量变化并快速更新 DB（而不是全量扫描）。
  - 风险：跨平台文件事件复杂、易漏；需要清晰的回退策略（定期 rescan）。
