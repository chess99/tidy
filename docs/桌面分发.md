# 桌面分发（Win/macOS）

本项目支持“普通用户无需 Node/Python 知识”的桌面分发形态：**Electron 桌面壳 + 本地后端/AI 子进程**，双击即可打开 UI。

## 形态与边界

- **桌面壳**：`desktop/`（Electron）负责窗口、端口选择、子进程启停、用户数据目录。
- **后端**：`server/` 在桌面模式下会托管 `client/dist`（静态文件 + SPA fallback），并提供 `/api/*`。
- **AI 服务**：`ai-service/` 通过本地 `http://127.0.0.1:<aiPort>` 提供推理（默认 8002）。

## 阶段化策略（当前）

- **阶段 1（小范围内测）**：不强依赖代码签名；更新采用“**提示式更新**”（提示有新版本 → 下载 → 引导用户手动覆盖安装）。
- **阶段 2（若公开分发）**：补齐 Win/macOS 签名与（macOS）notarize，再升级为“真自动更新”。

### 提示式更新（manifest 约定）

桌面壳支持通过环境变量 `TIDY_UPDATE_MANIFEST_URL` 指定一个 JSON manifest（https）：

```json
{
  "version": "0.1.1",
  "downloadUrl": "https://example.com/tidy-0.1.1.dmg",
  "notes": "修复：xxx"
}
```

当 `version` 高于当前版本时，会弹窗提示并打开 `downloadUrl`，用户手动覆盖安装。

## 开发者运行（桌面模式）

前置：本机有 Node；`ai-service` 需要可用的 Python 环境（开发期）。

```bash
npm run desktop:dev
```

可选环境变量：

- `TIDY_SERVER_PORT`：后端端口（默认 3001，冲突时自动回退）。
- `TIDY_AI_PORT`：AI 端口（默认 8002，冲突时自动回退）。
- `TIDY_DEV_UI_URL`：开发期直接加载 Vite dev server（例如 `http://127.0.0.1:5173`）。
- `TIDY_AI_BIN`：优先使用的 ai-service 可执行文件路径（若存在则无需 Python）。

## 数据目录

桌面版会把 DB/索引/缩略图等放在 Electron `userData` 下的 `data/`：

- Windows：`%APPDATA%/<app>/data/`
- macOS：`~/Library/Application Support/<app>/data/`

开发期为了方便，如果 userData 里还没有 DB，会 best-effort 从仓库根 `data/` 复制一次。

## 日志与问题反馈

桌面版会把运行日志写入用户数据目录下的 `logs/`（包含 `desktop.log`、`server.log`、`ai.log` 以及轮转出的 `.1/.2/...`）。

- macOS：`~/Library/Application Support/tidy-desktop/logs/`

用户遇到问题时：把该目录打包（zip）并附上当前版本号一起反馈即可。

### 日志上限（防爆炸）

- **单文件上限**：20MB（超过则轮转）
- **每类日志保留**：最多 5 份（`*.log` + `.1`…）
- **总目录上限**：200MB（超过则删除最旧文件）
桌面菜单里提供了：

- **Help → Open Logs Folder**：打开日志目录
- **Help → Copy Diagnostics**：复制版本/端口/日志路径等诊断信息
## 打包 ai-service 为可执行文件（桌面 sidecar）

桌面壳会优先尝试启动可执行的 `ai-service`（无需 Python）：
- 默认探测路径：`ai-service/dist/tidy-ai-service/` 下的可执行文件
- 或通过环境变量 `TIDY_AI_BIN` 指定可执行文件绝对路径

### macOS

```bash
cd ai-service
./scripts/build-ai-service.sh
```

### Windows

```bat
cd ai-service
scripts\\build-ai-service.bat
```

ai-service 打包脚本使用 `pip install --only-binary=:all:` 安装依赖，并在打包前运行
`pip check`。这可以让 Windows 分发构建在缺少 wheel 时直接失败，而不是要求用户或
构建机安装 C/C++ 编译工具。人脸能力依赖 `insightface==1.0.1`，不再排除 Windows。

## 打包桌面安装包（electron-builder）

前置：`client` 与 `server` 的依赖已安装，且 `client/dist` 与 `ai-service/dist/tidy-ai-service` 已生成。

```bash
cd client && npm run build
cd ../ai-service && ./scripts/build-ai-service.sh
cd ../desktop && npm install
cd ../desktop && npm run dist:mac
```

产物位于 `desktop/dist/`。
